# NewAPI Hugging Face Spaces 部署指南（源码编译版）

## 说明

本项目使用多阶段 Docker 构建：
1. **构建阶段**：使用 `golang:1.21-alpine` 编译源码
2. **运行阶段**：使用 `alpine:latest` 运行编译好的二进制文件

这样可以在 Hugging Face Spaces 上直接编译运行，无需本地预编译。

## 文件清单

| 文件 | 说明 |
|------|------|
| `Dockerfile` | 多阶段构建文件，从源码编译 |
| `README.md` | Hugging Face Spaces 配置文件 |

## 部署步骤

### 1. 注册 Hugging Face 账号

- 访问 https://huggingface.co
- 用 GitHub 账号一键登录

### 2. 创建 Space

1. 访问 https://huggingface.co/spaces/new
2. 填写信息：
   - **Space name**：自定义，如 `newapi`
   - **License**：选 `apache-2.0`
   - **Space SDK**：选择 **Docker**
   - **Space hardware**：CPU (free)
   - **Visibility**：Public（免费必须公开）
3. 点击 **Create Space**

### 3. 上传文件

创建后进入 Space 页面，选择 **Files** 标签：

1. 点击 **Add file** → **Upload files**
2. 上传本文件夹的 `Dockerfile` 和 `README.md`
3. 点击 **Commit changes to main**

或者使用 Git 命令行：

```bash
git clone https://huggingface.co/spaces/你的用户名/newapi
cd newapi
# 复制 Dockerfile 和 README.md 到该目录
git add .
git commit -m "Deploy NewAPI from source"
git push
```

### 4. 等待构建

上传后 Hugging Face 会自动构建 Docker 镜像。

**注意**：从源码编译需要较长时间（约 10-20 分钟），请耐心等待。

可以在 **Files → 构建日志** 中查看进度。

### 5. 访问服务

构建完成后，访问：

```
https://你的用户名-newapi.hf.space
```

首次访问会自动跳转到初始化页面，设置管理员账号密码。

## 配置外部数据库（强烈推荐）

### 为什么需要？

Hugging Face Spaces 免费版**没有持久化存储**，容器重启后所有数据丢失。

### 推荐：Neon 免费 PostgreSQL

1. 访问 https://neon.tech
2. 用 GitHub 登录（无需信用卡）
3. 创建项目，获取连接字符串：
   ```
   postgresql://用户名:密码@主机名.neon.tech/数据库名?sslmode=require
   ```

### 在 HF Spaces 中配置

1. 进入你的 Space 页面
2. 点击 **Settings** → **Variables and Secrets**
3. 点击 **New Secret**：
   - **Name**：`SQL_DSN`
   - **Value**：你的数据库连接字符串
4. 点击 **Save**
5. 重启 Space（在 Settings 里点击 **Restart this Space**）

### 数据库连接字符串格式

**PostgreSQL (Neon)**：
```
SQL_DSN=postgresql://用户名:密码@主机名.neon.tech/数据库名?sslmode=require
```

**MySQL**：
```
SQL_DSN=用户名:密码@tcp(主机地址:3306)/数据库名
```

## 保持服务在线

Hugging Face Spaces 免费版 **48 小时无访问会休眠**。

### 解决方法：UptimeRobot

1. 访问 https://uptimerobot.com
2. 注册免费账号
3. 点击 **Add New Monitor**：
   - **Monitor Type**：HTTP(s)
   - **Friendly Name**：NewAPI
   - **URL**：`https://你的用户名-newapi.hf.space`
   - **Monitoring Interval**：5 分钟（免费版最短）
4. 点击 **Create Monitor**

这样每 5 分钟会访问一次你的服务，防止休眠。

## 常见问题

### Q: 构建时间太长怎么办？

A: 从源码编译确实需要较长时间（10-20 分钟），这是正常的。Hugging Face Spaces 的构建环境资源充足，可以完成编译。

### Q: 构建失败怎么办？

A: 检查构建日志，常见原因：
- 网络问题导致依赖下载失败：重试即可
- 代码语法错误：检查是否有修改导致的问题

### Q: 如何更新 NewAPI 版本？

A: 修改 Dockerfile 中的源码版本，或重新 push 触发构建。

### Q: 可以部署多个 NewAPI 实例吗？

A: 可以，创建多个 Space 即可，但每个都需要单独配置数据库。

### Q: 如何备份数据？

A: 使用内置的数据导出功能（root 用户）导出 JSON 备份文件。
